## 概述

「系列文档」是 Firefly 博客的一项功能，用于将多篇相关文章组织为一个有章节导航的文档系列。每个系列在博客列表中显示为带有「系列」角标的卡片，内部有独立的侧边栏导航，支持章节间的前后跳转。

典型的应用场景：教材章节、课程讲义、系列教程、连载文章。

---

## 一、目录结构

每个系列是一个独立的文件夹，位于：

```
src/content/posts/Sets_of_doc/<系列名>/
```

文件夹内必须包含以下文件：

```
Sets_of_doc/<系列名>/
├── index.md          ← 系列首页（必选）
├── series.yaml       ← 系列配置文件（必选）
├── 01-第一章.md      ← 章节文件（可选，可以有多个）
├── 02-第二章.md
└── ...
```

各文件的职责：

| 文件 | 必选 | 说明 |
|------|------|------|
| `index.md` | ✅ | 系列首页，同时作为博客列表中的系列卡片。**文件名必须为 `index.md` 或 `index.mdx`** |
| `series.yaml` | ✅ | 系列配置，控制默认展示章节和外部同步 |
| `*.md` / `*.mdx` | 否 | 章节文件。排除 `index.md` 和 `series.yaml`，其余 `.md`/`.mdx` 文件均被识别为章节 |

---

## 二、`index.md` — 系列首页

`index.md` 有双重身份：

1. **系列首页**：访问 `/posts/Sets_of_doc/<系列名>/` 时显示的内容
2. **博客列表卡片**：在首页或归档页中作为普通文章卡片展示

### Frontmatter 参数

```yaml
---
title: "系列文档使用指南"       # 标题（必填）
published: 2026-05-28           # 发布日期
description: "..."               # 描述（显示在卡片和 SEO 中）
tags: [文档, 系列, 指南]         # 标签
category: "示例"                 # 分类
draft: false                     # 是否草稿
---
```

所有标准博客文章的 frontmatter 字段均可用：`image`、`pinned`、`lang`、`author` 等。

### 内容

正文使用标准 Markdown 编写，支持所有 Firefly 博客的 Markdown 扩展功能（表格、代码块、数学公式、Callout 提示框等）。

> **建议**：在首页中写入系列的简介、目录概览和学习指南，帮助读者快速了解该系列的内容结构。

## 三、`series.yaml` — 系列配置

`series.yaml` 控制系列的导航行为和外部同步。

### 完整配置

```yaml
# 默认展示章节（可选）
# 当用户访问系列首页时，侧边栏中默认高亮的章节 slug。
# slug 为文件名去掉扩展名，例如 "01-第一章"。
# 不设置此项时，首页不会高亮任何章节。
defaultChapter: "01-第一章"

# 外部同步（可选）
# 从外部目录自动同步章节文件到系列文件夹。
# 构建时根据文件修改时间（mtime）增量复制。
sync:
  source: "D:/MyDocuments/讲义/Chapter0"   # 源目录的绝对路径
```

### 参数说明

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `defaultChapter` | `string` | 无 | 系列首页侧边栏默认高亮的章节 slug。不设则不高亮 |
| `sync.source` | `string` | 无 | 外部章节文件的源目录路径（绝对路径）。不设则不启用同步 |

### 同步机制

当配置了 `sync.source` 后，每次 `astro build` 或 `astro dev` 启动时，`series-sync` 集成会：

1. 扫描源目录中的所有 `.md`/`.mdx` 文件（排除 `index.md` 和 `series.yaml`）
2. 对比源文件和目标文件的修改时间（`mtime`）
3. 仅复制源目录中更新过的文件到系列文件夹
4. 在终端输出同步日志：`[series-sync] "系列名": N copied, M skipped`

> **提示**：同步不会删除目标文件夹中的多余文件。如果源目录删除了某章节，需要手动清理目标文件夹。

## 四、章节文件

### 命名规则

章节文件按文件名前缀**自然排序**（`localeCompare` with `numeric: true`）。推荐使用数字前缀确保顺序：

```
01-第一章.md        ← slug: "01-第一章"
02-第二章.md        ← slug: "02-第二章"
03-第三章.md        ← slug: "03-第三章"
...
```

也支持子目录结构：

```
基础篇/
  01-概述.md       ← slug: "基础篇/01-概述"
  02-集合论.md     ← slug: "基础篇/02-集合论"
进阶篇/
  01-微积分.md     ← slug: "进阶篇/01-微积分"
```

### Frontmatter

章节文件**只需 `title` 字段**（用于侧边栏导航显示）：

```yaml
---
title: "第一章：入门指南"
---
```

`title` 会显示在：
- 桌面端侧边栏的章节列表中
- 移动端可折叠章节导航中
- 上一章 / 下一章导航按钮中

### 内容编写

章节内容通过 Astro 内容管线渲染，支持：

- **Markdown 基础语法**：标题、列表、引用、链接、图片
- **GFM 扩展**：表格、任务列表、删除线、脚注
- **LaTeX 数学公式**：行内 `$...$` 和块级 `$$...$$`（通过 KaTeX 渲染）
- **Obsidian Callout**：`:::note`、`:::warning`、`:::tip`、`:::important`、`:::caution`
- **Obsidian 高亮**：`==高亮文本==`
- **Obsidian Wiki 链接**：`[[页面名]]` 或 `[[页面名|别名]]`
- **代码块**：带语法高亮的围栏代码块
- **Mermaid 图表**：` ```mermaid ... ``` `

### 示例章节

```markdown
---
title: "第一章：入门指南"
---

## 第一节

这是第一章的内容。

### 数学公式

二次方程 $ax^2 + bx + c = 0$ 的求根公式：
$$x = \frac{-b \pm \sqrt{b^2 - 4ac}}{2a}$$

### 表格

| 符号 | 含义 | 示例 |
|------|------|------|
| $\mathbb{N}$ | 自然数集 | $\{0, 1, 2, \dots\}$ |
| $\mathbb{Z}$ | 整数集 | $\{\dots, -2, -1, 0, 1, 2, \dots\}$ |

### 提示框

:::tip
这是一个提示框，用于补充说明。
:::

:::warning
注意：DSE 考试中务必写清楚演算步骤！
:::
```

## 五、页面结构与导航

### 桌面端布局

进入系列文档后，页面布局为：

```
┌─────────────────────────────────────────────────────┐
│  [公告]                    │  文档内容   │ [站点统计] │
│  ─────────                │            │ [日历]     │
│  [系列章节导航]            │            │            │
│    ├ 第一章 ← 当前高亮     │            │            │
│    ├ 第二章                │            │            │
│    └ 第三章                │            │            │
│  ─────────                │            │            │
│  [音乐播放器]              │            │            │
└─────────────────────────────────────────────────────┘
```

- **左侧栏**：固定显示公告、章节导航、音乐播放器。章节过多时自动滚动
- **中间**：文档正文，带上一章 / 下一章导航按钮
- **右侧栏**：保留站点统计和日历组件

### 移动端

- 章节导航折叠在可展开的 `<details>` 元素中
- 文档内容全宽显示

### URL 结构

```
/posts/Sets_of_doc/<系列名>/              ← 系列首页（显示 index.md）
/posts/Sets_of_doc/<系列名>/<章节slug>/   ← 章节页面
```

例如：
- `/posts/Sets_of_doc/sample-series/` → 本指南
- `/posts/Sets_of_doc/sample-series/01-第一章/` → 第一章

### 导航

- **返回系列首页**：侧边栏顶部的链接，指向系列首页
- **章节列表**：点击任意章节跳转
- **上一章 / 下一章**：正文底部的前后导航按钮

## 六、创建新系列的步骤

### 1. 创建文件夹

在 `src/content/posts/Sets_of_doc/` 下新建文件夹：

```
Sets_of_doc/my-tutorial/
```

### 2. 创建 `index.md`

```yaml
---
title: "我的教程系列"
published: 2026-06-01
description: "这是一个系列教程"
tags: [教程]
category: "学习"
draft: false
---

欢迎来到我的教程系列！本系列将带你从零开始学习...
```

### 3. 创建 `series.yaml`

```yaml
defaultChapter: "01-简介"
```

### 4. 创建章节文件

```markdown
---
title: "第一章：简介"
---

## 概述

这是第一章的内容...
```

### 5. 构建验证

```bash
pnpm astro build
# 或开发模式
pnpm astro dev
```

访问 `/posts/Sets_of_doc/my-tutorial/` 查看效果。

## 七、高级用法

### 外部目录同步

适用于章节内容在别处维护（如 Obsidian vault、Git 子模块）的场景：

```yaml
# series.yaml
sync:
  source: "C:/Users/xuless/Documents/讲义/Chapter0"
```

源目录下的 `.md` 文件会被自动复制到系列文件夹，无需手动管理。

### 子目录章节

章节可以按主题分文件夹组织：

```
Sets_of_doc/tutorial/
├── index.md
├── series.yaml
├── 基础/
│   ├── 01-环境搭建.md
│   └── 02-第一个程序.md
└── 进阶/
    ├── 01-数据结构.md
    └── 02-算法分析.md
```

slug 格式为 `子目录/文件名`（去掉扩展名），如 `基础/01-环境搭建`。

### 草稿章节

将章节文件的 `draft` 设为 `true`：

```yaml
---
title: "未完成的章节"
draft: true
---
```

草稿章节不会在构建产物中出现，但在开发模式下可见。

## 八、常见问题

**Q: 系列首页为什么不显示 `index.md` 的内容？**

A: 检查 `series.yaml` 是否存在，以及 `index.md` 的 frontmatter 是否完整（至少需要 `title` 字段）。

**Q: 章节排序不对？**

A: 章节按文件名字符串自然排序（`localeCompare` with `numeric: true`）。推荐使用 `01-`、`02-` 格式的前缀。

**Q: 如何隐藏某个章节？**

A: 将章节文件的 `draft` 设为 `true`，或在文件名前加下划线（如 `_hidden-chapter.md`），此类文件不会被 `collectChapters()` 识别——不对，目前 `collectChapters` 没有排除下划线前缀。建议使用 `draft: true`。

**Q: 为什么右侧栏消失了？**

A: 这通常是已知的布局 bug。如果遇到，请在终端查看是否有 JavaScript 错误。完整的系列文档页面应保留右侧栏的站点统计和日历。如果确实缺失，请检查浏览器控制台。
